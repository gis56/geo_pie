=======================
Зоны санитарной охраны.
=======================

.. _String : https://qgis.org/pyqgis/3.34/core/QgsFieldProxyModel.html#
   qgis.core.QgsFieldProxyModel.String
.. _Double : https://qgis.org/pyqgis/3.34/core/QgsFieldProxyModel.html#
   qgis.core.QgsFieldProxyModel.Double

.. .. contents:: Содержание
..   :depth: 2

Отрисовка зон санитарной охраны элептической и квадратной формы по заданным 
радиусам и азимуту.

Подготовка данных
-----------------

Необходимо создать точечный слой ``PointGeometry``. Объекты этого слоя будут
центрами зон. Вычисления возможны только в спроектированных системах координат.
Если используемый точечный слой в географических (градусных) координатах, то
для расчетов может быть взята проекция проекта. Если и слой и проект 
не спроецированны, то вычисления производится не будут.

.. important::
   Либо слой, либо проект должен быть в плоской проекции.

В аттрибутивной таблице точечного слоя  должны присутствовать следующие поля:

.. table:: Структура атрибутивной таблицы

    +----------+-----------+--------------------+------------+
    | Поле     | Тип поля  | Описание           | Зона       |
    +==========+===========+====================+============+
    | ``r1``   | `Double`_ | Сторона квадрата   | Первая     |
    +----------+-----------+--------------------+------------+
    | ``r_n2`` | `Double`_ | "Северный" радиус  | Вторая     |
    +----------+-----------+--------------------+            |
    | ``r_s2`` | `Double`_ | "Южный" радиус     |            |
    +----------+-----------+--------------------+            |
    | ``r_w2`` | `Double`_ | "Западный" радиус  |            |
    +----------+-----------+--------------------+            |
    | ``r_o2`` | `Double`_ | "Восточный" радиус |            |
    +----------+-----------+--------------------+------------+
    | ``r_n3`` | `Double`_ | "Северный" радиус  | Третья     |
    +----------+-----------+--------------------+            |
    | ``r_s3`` | `Double`_ | "Южный" радиус     |            |
    +----------+-----------+--------------------+            |
    | ``r_w3`` | `Double`_ | "Западный" радиус  |            |
    +----------+-----------+--------------------+            |
    | ``r_o3`` | `Double`_ | "Восточный" радиус |            |
    +----------+-----------+--------------------+------------+

Если отсутствуют все поля для какого либо пояса, зона этого пояса не создаётся.
Если хотя бы в одном поле пояса встречается значение ``0`` или ``NULL``, 
то зона этого пояса тоже не создаётся.
Если поля не были созданы вручную в атрибутивной таблице слоя, то при 
выполнении модуля будет предложено добавить поля автоматически. В этом случае
будут добавлены поля всех радиусов со значениями ``NULL``

Ввод данных.
------------

В меню **QGis** выбрать пункт 
:menuselection:`Вектор  --> Мои скрипты --> Зоны СО`.
Откроется диалоговое окно:

.. image:: /image/zso.png
   :alt: Зона СО
   :height: 360px
   :width: 422px
   :scale: 80 %
   :align: center

В диалоговом окне выбрать слой со скважинами, выбрать поле с названиями скважин
и определить угол азимута на который будет развернуты все зоны.
Зоны будут повернуты по часовой стрелке на указанный угол от 
северного направления при положительном значении и 
против часовой при отрицательном.
При наличии выбранных объетов, опрецию можно применить только 
к выбранным объектам.
Для этого необходимо поставить "галочку" 
:guilabel:`Только выбранные объекты`.
Если в атрибутивной таблице не создана структура полей для радиусов, 
то будет предложено добавить их автоматически. После создания структуры
необходимо внести значения и запустить модуль снова.

Полученные результаты.
----------------------

* Если в атрибутивной таблице не нашлось нужных полей, то в результате работы
  плагина будут добавлены поля для всех радиусов.
* В случае когда все данные были заполнены будет создана группа слоев:
    * слой полигонов зон СО
    * линейный слой радиусов зон СО
    * если слой скважин был в географических координатах, будет создан
      точечный слой углов первой зоны СО с координатами в атрибутивной
      таблице

.. note:: Полученные слои будут в системе координат исходного слоя
